1 Business Requirements
	1.1 Background

	У великих бізнес-центрах, кампусах і громадських просторах паркування є складним завданням через високу кількість автомобілів, різноманітні категорії користувачів (співробітники, орендарі, відвідувачі), а також обмеженість ресурсів. Також це все потребує достатнього адміністрування.

	1.2 Business Opportunities

     Parkopedia:
     Функціонал: Це глобальна платформа для пошуку паркомісць, яка надає інформацію про доступність паркувальних місць в реальному часі. Вона має інтеграцію з GPS-навігацією і дозволяє користувачам переглядати тарифи, час паркування та доступність місць.
     Переваги: Велика база даних паркувальних зон по всьому світу, зручний інтерфейс для користувачів.
     Недоліки: Обмежена можливість інтеграції з місцевими системами для адміністративного управління, відсутність гнучкості у тарифікації.
     SpotHero:
     Функціонал: Платформа для бронювання паркувальних місць у великих містах США. SpotHero пропонує мобільний додаток, через який користувачі можуть забронювати паркувальне місце заздалегідь.
     Переваги: Інтуїтивно зрозумілий додаток для користувачів, інтеграція з великими паркінгами та парковками в міських зонах.
     Недоліки: Зосереджено лише на великих містах США, що обмежує застосування на міжнародному ринку. Відсутність гнучкої настройки для адміністраторів.
2 Vision of the Solution
	2.1 Vision Statement

     Система управління паркуванням покликана забезпечити ефективне, прозоре та автоматизоване керування паркомісцями для великих бізнес-центрів, кампусів та інших інфраструктурних об’єктів. Основна мета — спростити адміністративні процеси, покращити користувацький досвід та підвищити ефективність використання доступних ресурсів. Система дозволить адміністраторам швидко налаштовувати доступ до паркомісць для різних груп користувачів (наприклад, викладачів, студентів, відвідувачів), а кінцевим користувачам — легко знаходити вільні місця через мобільний додаток. Завдяки інтеграції з IoT-пристроями, такими як ультразвукові датчики, система забезпечить точне відстеження зайнятості паркомісць у реальному часі, що зменшить хаос і конфлікти, пов’язані з паркуванням.
     
	2.2 Major Features
	
	Функції для адміністраторів:
     1. Управління паркомісцями:
* Додавання, видалення та редагування паркувальних місць.
     2. Моніторинг паркомісць.
* Інтеграція з датчиками для відстеження зайнятості місць.
     3. Бронювання для орендарів:
* Надання компаніям можливості резервувати місця.
     Функції для кінцевих користувачів:
     1. Перегляд паркомісць
     2. Пошук доступних місць:
* Відображення вільних місць.
* Фільтри для пошуку.
     Інтеграція з обладнанням:
     1. Підключення датчиків зайнятості місць:
* Підтримка сучасних IoT-рішень для збору даних про стан паркомісць.

3 Scope and Limitations
     3.1 Scope of Initial Release

	Функціонал серверної частини:
     База даних:
* Інформація про паркувальні місця: Дані про всі паркувальні місця, їх доступність, типи.
* Дані користувачів, а також права доступу для різних категорій паркомісць.
* Об’єднання користувачів за групами (викладачі, лікарі, студенти)
* Можливість зробити бекап бази даних
     API для взаємодії з клієнтами:
* Аутентифікація та авторизація.
* API для пошуку паркувальних місць.
* Моніторинг та оновлення статусу місць.
     Інтеграція з IoT пристроями:
* Збір даних з пристроїв.
     Функціонал IoT частини: 
* Датчики для визначення наявності місця.

     
     3.2 Scope of Subsequent Releases

     Основні функції мобільного клієнта:
     Реєстрація та авторизація користувачів:
* Відповідає за зручну реєстрацію через мобільний телефон за допомогою електронної пошти.

     Перегляд паркувальних місць:
* Можливість фільтрувати місця за критеріями ( доступність для електромобілів, парковка для інвалідів тощо).
     Підтримка багатомовності:
* Мобільний додаток може підтримувати кілька мов для зручності користувачів з різних країн.
     
     Основні функції веб-клієнта:
     Авторизація адміністратора:
* Спеціальний процес авторизації, який дозволяє тільки уповноваженим користувачам мати доступ до адміністрування.
     Управління паркувальними місцями:
* Додавання нових паркувальних місць: Адміністратор може додавати нові паркувальні місця в систему, визначати їх типи (стандартні, для електромобілів, для інвалідів тощо), статус (вільне/заброньоване).
* Редагування існуючих місць: Можливість змінювати інформацію про вже наявні паркувальні місця, такі як їх тип або доступність.
* Видалення паркувальних місць: Адміністратор може видаляти паркувальні місця, які більше не використовуються.
     
     
     Налаштування системи:
* Активування або деактивування доступ різних груп користувачів до паркомісць.

     Основні функції IoT: 
* Ультразвукові датчики для визначення дистанції.

4 Business Context
     4.1 Operationg Enviroment

     Для виконання проекту будуть використовуватися:
     WEB розробка: 
     * HTML, CSS, JavaScript — для створення інтерактивного веб-інтерфейсу для адміністраторів. 
     * Node.js — для створення серверної частини з API для взаємодії з клієнтами та IoT-пристроями.
     * Express.js — як фреймворк для спрощення розробки API
MOBILE розробка: 
     * Kotlin — для створення мобільного додатка на Android, що забезпечить зручний інтерфейс для користувачів.
     СУБД: 
     * PostgreSQL — для зберігання даних про паркувальні місця, користувачів і їхні права доступу.
     * pgAdmin — для зручного адміністрування бази даних під час розробки.
     
     
     ІоТ: 
     * Ультразвукові датчики (HC-SR04) для визначення дистанції до автомобілів. 
     * ESP32 — як платформа для IoT-пристроїв із підтримкою Wi-Fi та MQTT.
     * MQTT-протокол — для передачі даних від IoT-пристроїв до сервера.
